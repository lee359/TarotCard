# 月之秘語卡牌規則

本文件記錄目前網站的塔羅占卜流程、抽牌方式、正逆位判定、翻牌限制與結果呈現規則。

## 1. 開始占卜條件

使用者必須同時完成以下操作，才可按下「開始抽牌」：

1. 輸入占卜問題，最多 120 個字。
2. 明確點選一個占卜主題。

目前提供四個主題：

- 感情
- 事業
- 自我
- 今日指引

若問題或主題尚未填寫，「開始抽牌」按鈕維持停用狀態。

開始占卜時，首頁會將以下資料傳至 `/reading`：

```text
topic     選擇的占卜主題
question  使用者輸入的問題
flow      本次占卜流程的唯一識別碼
```

## 2. 牌組來源

`reading.vue` 會優先讀取 Firestore 的 `tarotCards` collection。

- Firestore 至少有 3 張牌時，使用遠端牌組。
- Firestore 無法讀取或牌數少於 3 張時，使用頁面內建牌組。

每張牌包含以下基本資料：

```text
number    卡牌編號
name      中文名稱
english   英文名稱
symbol    卡牌符號
keywords  關鍵字
meaning   基本牌義
```

## 3. 洗牌與抽牌

每次開始占卜或按下「重新抽牌」時：

1. 複製目前牌組，避免直接修改原始資料。
2. 使用 Fisher–Yates 演算法隨機洗牌。
3. 取洗牌後的前 3 張牌。
4. 同一次抽牌不會出現重複卡牌。
5. 重設所有翻牌狀態並關閉結果視窗。

三張牌依序代表：

| 順序 | 牌位 | 意義 |
| --- | --- | --- |
| 1 | 過去・根源 | 問題的背景、起點或既有影響 |
| 2 | 現在・課題 | 當下狀態及需要面對的課題 |
| 3 | 未來・指引 | 後續發展與可參考的行動方向 |

## 4. 正位與逆位

每張抽出的牌會獨立判定正逆位：

```text
Math.random() < 0.5  → 逆位
其他情況             → 正位
```

因此每張牌的正位與逆位機率各為 50%，三張牌彼此不互相影響。

- 正位：顯示原始牌面方向與基本牌義。
- 逆位：牌面上下旋轉 180 度，並以「能量受阻、延遲或轉向內在」作為目前的通用解讀框架。

正逆位只屬於當次抽牌的執行期狀態，不會寫入或修改 Firestore 的 `tarotCards` 資料。

## 5. 翻牌順序

卡牌必須由左至右依序翻開：

1. 第一張牌可直接翻開。
2. 第一張牌翻開後，第二張牌才會解鎖。
3. 第二張牌翻開後，第三張牌才會解鎖。

提前點擊尚未解鎖的卡牌時：

- 卡牌會播放晃動動畫。
- 第二張牌提示「要了解現在，得先回想過去」。
- 第三張牌提示「欲知曉未來，要先理解現在」。
- 不會改變該牌的翻牌狀態。

卡牌翻開後不可重複翻牌。

## 6. 卡牌說明

翻牌後，每張牌下方會顯示：

- 正位或逆位標示。
- 該牌的基本牌義。
- 逆位時額外加入能量受阻或內化的提示。

卡牌方向、牌義標示及結果視窗使用同一份 `reversed` 狀態，避免畫面與文字判定不一致。

## 7. 查看結果

三張牌全部翻開後才會出現「查看結果」按鈕。

結果視窗包含：

- 使用者原始問題。
- 本次占卜主題。
- 未來牌的名稱及正逆位狀態。
- 過去、現在、未來三張牌的整體串聯文字。
- 每張牌的牌位、名稱、正逆位與個別摘要。
- 占卜肯定語。

使用者可關閉結果視窗、重新抽牌，或返回首頁更換主題。

## 8. 重新抽牌

按下「重新抽牌」後：

- 沿用目前問題與主題。
- 重新洗牌並抽取 3 張牌。
- 重新判定每張牌的正逆位。
- 重設翻牌與結果視窗狀態。
- 不會再次增加本次流程的主題完成次數。

## 9. 主題完成次數判定

只有同時符合以下條件，該主題的完成次數才會增加 1：

1. 使用者在首頁明確點選主題。
2. 使用者輸入問題並按下「開始抽牌」。
3. 使用者在 `reading.vue` 依序翻開三張牌。
4. 使用者按下「查看結果」。
5. URL 中的 `flow` 與 `sessionStorage` 保存的流程資料一致。

每個 `flow` 最多計數一次。成功寫入 Firestore 後會移除該流程資料，快速重複點擊「查看結果」也不會重複計數。

統計寫入 Firestore：

```text
siteStats/topicSelections
├─ love    感情完成次數
├─ career  事業完成次數
├─ self    自我完成次數
├─ daily   今日指引完成次數
└─ lastCompletedAt  最後完成時間
```

## 10. 目前限制

- 逆位使用共用解讀框架，尚未為每張牌建立獨立逆位牌義。
- 目前固定使用三張牌的「過去／現在／未來」牌陣。
- 問題會透過 URL query 傳遞，不適合輸入敏感個人資料。
- 塔羅結果僅供自我探索與娛樂，不應取代醫療、法律、財務或其他專業建議。
